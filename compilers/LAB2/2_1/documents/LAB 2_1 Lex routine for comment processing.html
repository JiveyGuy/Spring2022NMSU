LAB 2_1 Lex routine for comment processing

  • Due Sunday by 11:59pm
  • Points 10
  • Submitting a file upload
  • File Types zip

lab2_1

In this lab, you are to make some simple changes to the LEX program provided HERE Download HERE

You need to test your code with the input from HERE Download HERE

 

VIDEO DESCRIPTION OF ACTIVITY Download VIDEO DESCRIPTION OF ACTIVITY


The provided program performs the following when given a "/*  */" commented  input

1)  The program removes all comments from the file that have the "/*  ... */  format
2)  When put into debug mode, it will report an error that there may be embedded comments

 

You are to experiment with the code.  Figure out how to get the error messages to tell you that there are embedded comments, and then report the line number where the error occurred.  Please note the following

a)  lines start at line 1 not 0

b)  the provided starting point prints out new line.  When you program to notice new line, you will find your output messed up.  You will need to figure out how to generate a new line in those cases.

c) make sure you get rid of any compiler warnings

 

Learning Objectives:

  1. Reinforce the use of LEX
  2. Introduce a new LEX directive to count lines
    1. We will need to know which line we are on when we generate an error later
  3. Reinforce the use of global variables to add in keeping global information (line count)
  4. Learn how if statements are controlled by variables for output control
  5. Learn about how lex directives can consume input data.

 

 

Your task is to download the file and add a new variable to count the lines of the input. 

Additionally, Where-ever there is an error message in the LEX program (in the C portions of the right hand side action), you are to add to the error message to include the line number.

The character of newline should be known to you from your use of "printf()" in your C++ programming experience

You MUST create a LAB2_1 folder.  You MUST have a makefile (with documentation).

 

Deliverables

ZIP FILE

1)  A correct , documented makefile with executable lab2_1 as target

2)   Header comment in your LEX file which describes what changes you made, where and how 

3)  Inline comments for any new LEX actions

4)  A screen shot of your output

 

 

This project mainly emphasizes your use of LEX, and how well you comment.  Your Makefile must have header comments Your target MUST be lab2_1
